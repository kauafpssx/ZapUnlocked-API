---
title: "Marcar como Lido"
icon: "check-double"
api: "POST /messages/read"
---

<ParamField header="x-api-key" type="string" required placeholder="SUA_CHAVE_SECRETA">
  Chave de autenticação da API.
</ParamField>

Marca uma ou mais mensagens como lidas, enviando o duplo check azul para o remetente.

### Corpo da Requisição

<ParamField body="phone" type="string" required>
  Número do remetente/chat com código do país (ex: `5511999999999`).
</ParamField>

<ParamField body="messageIds" type="array" required>
  Lista de IDs das mensagens a marcar como lidas (com `type: "id"`), ou textos exatos (com `type: "text"`).
  ```json
  ["3EB0ABCDEF123456", "3EB0ABCDEF123457"]
  ```
</ParamField>

<ParamField body="type" type="string" default="id">
  Como localizar as mensagens: `"id"` (pelo ID) ou `"text"` (pelo conteúdo).
</ParamField>

<Note>
  Este endpoint só funciona se os confirmações de leitura estiverem ativas nas configurações de privacidade do bot (`readReceipts: "all"`).
</Note>


<ResponseField name="success" type="boolean">
  Indica se a operação foi aceita com sucesso.
</ResponseField>

<ResponseField name="message" type="string">
  Mensagem de status.
</ResponseField>

<ResponseExample>

```json 200 Sucesso
{
  "success": true,
  "message": "1 message(s) marked as read."
}
```

```json 404 Não encontrado
{
  "success": false,
  "message": "Message not found."
}
```

</ResponseExample>
