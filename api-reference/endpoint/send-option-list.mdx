---
title: "Lista de Opções"
icon: "list-ul"
api: "POST /messages/send-option-list"
---

<ParamField header="x-api-key" type="string" required placeholder="SUA_CHAVE_SECRETA">
  Chave de autenticação da API.
</ParamField>

Envia uma mensagem com uma lista de opções. O usuário abre a lista tocando no botão e seleciona um item. A seleção dispara um evento `message.list_reply` via webhook.

### Corpo da Requisição

<ParamField body="phone" type="string" required>
  Número do destinatário com código do país (ex: `5511999999999`).
</ParamField>

<ParamField body="message" type="string">
  Texto da mensagem exibido acima da lista.
</ParamField>

<ParamField body="title" type="string">
  Título da lista (exibido no cabeçalho do painel de seleção).
</ParamField>

<ParamField body="buttons" type="array" required>
  Lista de opções. Cada item representa uma linha selecionável:
  ```json
  [
    { "id": "prod_01", "text": "Camiseta", "description": "P, M, G" },
    { "id": "prod_02", "text": "Calça", "description": "Tamanhos 38-46" }
  ]
  ```
  Quando o usuário seleciona um item, o `id` é retornado no evento de webhook `message.list_reply`.
</ParamField>

<Note>
  O `id` de cada item é retornado no webhook quando o usuário seleciona. Use-o para identificar a escolha sem banco de dados.
</Note>


<ResponseField name="success" type="boolean">
  Indica se a lista foi enviada com sucesso.
</ResponseField>

<ResponseField name="message" type="string">
  Mensagem de status.
</ResponseField>

<ResponseExample>

```json 200 Sucesso
{
  "success": true,
  "message": "Interactive message sent."
}
```

```json 401 Não autorizado
{
  "error": "Unauthorized",
  "message": "Invalid or missing API key. Send via x-api-key header (preferred) or ?API_KEY= query param."
}
```

</ResponseExample>
