---
title: "Botões de Ação"
icon: "bolt"
api: "POST /messages/send-button-actions"
description: "Envia uma mensagem com botões de resposta rápida."
---

<ParamField header="x-api-key" type="string" required placeholder="SUA_CHAVE_SECRETA">
  Chave de autenticação da API.
</ParamField>

Envia botões do tipo **ação** (resposta rápida). Aparecem como chips abaixo da mensagem. Ao clicar, geram um evento `message.button_reply` com o ID e texto do botão.

Os botões são **stateless** — as instruções de destino ficam criptografadas no ID do botão.

### Corpo da Requisição

<ParamField body="phone" type="string" required>
  Número do destinatário com código do país (ex: `5511999999999`).
</ParamField>

<ParamField body="message" type="string">
  Texto principal exibido acima dos botões.
</ParamField>

<ParamField body="buttons" type="array" required>
  Lista de botões de ação. Cada item:

  <Expandable title="Estrutura do botão">
    <ParamField body="id" type="string">
      ID único do botão (gerado automaticamente se omitido).
    </ParamField>
    <ParamField body="text" type="string" required>
      Texto exibido no botão.
    </ParamField>
    <ParamField body="webhook" type="object">
      Configuração do webhook disparado ao clicar:
      - `url`: URL de destino
      - `method`: Método HTTP (padrão `POST`)
      - `headers`: Cabeçalhos personalizados
      - `body`: Corpo personalizado (suporta placeholders de webhook como `{{from}}`, `{{text}}`, `{{timestamp}}`, etc.)
    </ParamField>
    <ParamField body="reaction" type="string">
      Emoji para reagir automaticamente ao clique.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="reply" type="string">
  ID ou texto da mensagem para responder.
</ParamField>

<ParamField body="type" type="string" default="id">
  Como o campo `reply` é interpretado: `id` (padrão) ou `text`.
</ParamField>
