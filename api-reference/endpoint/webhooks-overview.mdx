Registre múltiplos endpoints externos para receber eventos do WhatsApp em tempo real. Cada webhook tem um **nome único** (ex: `meu-crm`, `meu-bot`) e pode ser inscrito em tipos de eventos diferentes de forma independente.

<CardGroup cols={3}>
  <Card title="CRUD completo" icon="pen-to-square">
    Criar, listar, atualizar, deletar, ativar/desativar e testar cada webhook individualmente.
  </Card>
  <Card title="Eventos granulares" icon="filter">
    Inscreva cada webhook só nos eventos que importam. Use `"*"` para receber tudo.
  </Card>
  <Card title="Body personalizado" icon="code">
    Defina um payload customizado com placeholders dinâmicos por webhook.
  </Card>
</CardGroup>

---

## <Icon icon="envelope" iconType="sharp-regular" color="#9d4fe6" size={22} /> Envelope Padrão

Todos os eventos entregues seguem a mesma estrutura:

```json
{
  "event": "message.text",
  "timestamp": "2026-05-28T02:00:00Z",
  "data": { ... }
}
```

Se o webhook tiver `body` personalizado, os placeholders (`{{from}}`, `{{text}}`, etc.) são substituídos antes do envio. Sem `body`, o envelope padrão é enviado completo.

## <Icon icon="list" iconType="sharp-regular" color="#9d4fe6" size={22} /> Tipos de Eventos

## <Icon icon="brackets-curly" iconType="sharp-regular" color="#9d4fe6" size={22} /> Placeholders no Body

Se o webhook tiver `body` personalizado, você pode usar:

| Placeholder | Descrição |
| :---------- | :-------- |
| `{{from}}` | Número de quem enviou |
| `{{name}}` | Nome do contato |
| `{{text}}` | Texto da mensagem |
| `{{phone}}` | Mesmo que `from` |
| `{{id}}` | ID da mensagem |
| `{{timestamp}}` | Data/hora do evento |
| `{{event}}` | Tipo do evento |

Se `body` estiver vazio, o envelope padrão é enviado automaticamente.
