## <Icon icon="cloud" iconType="sharp-regular" color="#9d4fe6" size={22} /> Compatibilidade com Alwaysdata (Plano Free)

<CardGroup cols={2}>
  <Card title="RAM: 256 MB" icon="microchip">
    Consumo idle de ~140 MB deixa uma margem confortável. Picos de operação ficam em ~220 MB.
  </Card>
  <Card title="CPU: 0,25 vCPU" icon="gauge-high">
    Otimizado para uso fracionado. Picos ocorrem apenas durante conversão de áudio ou imagem.
  </Card>
  <Card title="Armazenamento: 1 GB SSD" icon="hard-drive">
    Generoso para uso normal. A compressão gzip reduz o histórico em até 70%.
  </Card>
  <Card title="Backups: 3 dias" icon="clock-rotate-left">
    Retenção automática de 3 dias pelo Alwaysdata. Sessão de autenticação armazenada em arquivo local.
  </Card>
</CardGroup>

<Warning>
  No plano gratuito do Alwaysdata, processos iniciados via SSH são encerrados após inatividade. Para manter a API rodando 24/7, registre a execução em **Web > Serviços** no painel do Alwaysdata.
</Warning>

## <Icon icon="circle-check" iconType="sharp-regular" color="#9d4fe6" size={22} /> Boas Práticas

- **Limpeza de histórico**: use `/management/cleanup` regularmente para evitar acúmulo de dados no disco
- **Mídias pesadas**: a fila de processamento garante que apenas um arquivo seja processado por vez, sem picos de memória
- **Log em produção**: mantenha o log no modo normal para evitar escritas excessivas em disco

<Warning>
  Se o consumo de RAM exceder o limite do host, o processo pode ser encerrado pelo OOM Killer do SO. A arquitetura Python + Neonize é significativamente mais eficiente que implementações Node.js.
</Warning>
