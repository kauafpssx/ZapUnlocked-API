A **ZapUnlocked-API** foi projetada para rodar de forma estável mesmo em ambientes com recursos altamente limitados. Cada componente é otimizado para estabilidade e desempenho.

---

## <Icon icon="layer-group" iconType="sharp-regular" color="#9d4fe6" size={22} /> Fila de Mídia

Para evitar que múltiplos uploads simultâneos excedam os limites de recursos, a API usa um sistema de **Fila Inteligente**:

<Steps>
  <Step title="Recebimento" icon="inbox">
    A API recebe múltiplas requisições de upload de mídia simultâneas.
  </Step>
  <Step title="Enfileiramento" icon="list-ol">
    Em vez de baixar tudo ao mesmo tempo, os arquivos entram em uma fila de espera ordenada.
  </Step>
  <Step title="Processamento um por vez" icon="play">
    O sistema processa um arquivo por vez, evitando picos de uso de memória e disco.
  </Step>
  <Step title="Limpeza automática" icon="trash">
    Assim que o arquivo é enviado ao WhatsApp, ele é imediatamente deletado do servidor e o espaço é liberado.
  </Step>
</Steps>

<Info>
  Este sistema garante que arquivos de até **2 GB** (via protocolo) possam ser enviados mesmo com recursos de servidor limitados.
</Info>

## <Icon icon="file-zipper" iconType="sharp-regular" color="#9d4fe6" size={22} /> Compressão de Dados

Todos os **históricos de mensagens** e **índices de contatos** são armazenados em formato **JSON comprimido (gzip)**.

<Steps>
  <Step title="Armazenamento" icon="floppy-disk">
    Quando uma mensagem é salva, o JSON é comprimido antes de ser escrito no disco.
  </Step>
  <Step title="Leitura" icon="magnifying-glass">
    Ao buscar mensagens, o arquivo é descomprimido em memória, processado e retornado.
  </Step>
  <Step title="Economia" icon="chart-line">
    A compressão reduz o tamanho do arquivo em até **70%**, permitindo armazenar mais dados no mesmo espaço.
  </Step>
</Steps>

<Tip>
  Para detalhes sobre limites de armazenamento e cálculos de consumo, veja a página de [Armazenamento de Mensagens](/api-reference/message-storage).
</Tip>

## <Icon icon="gauge-high" iconType="sharp-regular" color="#9d4fe6" size={22} /> Desempenho de Compressão

A compressão não prejudica as requisições. Mesmo com o ciclo descomprimir-processar-comprimir, as latências permanecem baixas.

**Latências reais medidas** (servidor na Virgínia/EUA, cliente no Brasil com ~160ms de ping):

| Endpoint | Tempo de Resposta |
| :--- | :--- |
| `GET /management/volume_stats` | ~1.082 segundos |
| `POST /management/fetch_messages` | ~956 ms |
| `POST /management/recent_contacts` | ~959 ms |

<Info>
  A maior parte do tempo é consumida pela latência de rede, não pela compressão. O overhead do gzip é mínimo e compensado pela economia de espaço.
</Info>
